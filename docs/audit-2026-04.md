# CaliLean Repo Audit — Brand-Coupling Inventory & Order of Operations

> Author: CTO · Date: 2026-04-26 · Branch: `master` · Last commit: `2182f45`
> Scope: SKA-3 (E1: Repo audit + brand-coupling inventory)
> Companion docs: `.project-state/audit.md` (2026-04-22 stack snapshot), `.project-state/health.md`, `.project-state/roadmap.md`

## TL;DR

The repo is a working Medusa 2.13.6 backend + Next.js 15 storefront, currently branded end-to-end as **Bluum Peptides**. The brand is encoded in **30 files / ~182 occurrences** across UI copy, design tokens, CSS variables, Tailwind theme keys, an entire `modules/bluum/` folder, the seed catalog, image CDN URLs (`bluumpeptides.com`), and SEO metadata. A clean swap to CaliLean is mostly mechanical (tokens + folder rename + copy + image hosting), but two surfaces are non-trivial: **(1) the product image CDN** (56+ hard-coded `bluumpeptides.com` URLs in seed JSON + a `next.config.js` allowlist) and **(2) the support email** (`hello@bluumpeptides.com`). Compliance — peptides require Rx in CA — is not addressed anywhere in the codebase and is the single largest gap blocking storefront launch.

## 2. Build & Run — Verified

The README's commands match the actual `package.json` scripts:

- Backend: `pnpm install` → `cp .env.template .env` → `pnpm ib` (init: migrations + seed) → `pnpm dev`. Build: `medusa build && node src/scripts/postBuild.js`.
- Storefront: `pnpm install` → `cp .env.local.template .env.local` → `pnpm dev` (waits for backend then launches). Build: `pnpm build:next`.
- Email preview: `pnpm email:dev` on backend → `localhost:3002`.
- E2E: `pnpm test-e2e` in storefront (Playwright; 10 specs in `storefront/e2e/`).
- Seed: `python3 scripts/seed-products.py` (loads `storefront/src/data/products-seed.json`, posts to Medusa Admin API).

I did **not** run a full `pnpm install + build` end-to-end this heartbeat — services not started in this environment. CI (see §6) runs lint + build for both apps on every push to `master` and is the proxy for green.

## 5. Recommendation — Order of Operations for the Brand Swap (SKA-4 input)

I recommend three phases, sequenced so the storefront stays green at every step.

### Phase 0 — Compliance gate (escalate to CEO)

Before any branding work, lock the legal path. Peptides for human use require a prescriber in CA; "research use only" copy is the current escape hatch but contradicts a consumer storefront positioning. Two outcomes are possible: **(a)** keep "research-use-only" framing and ship without telehealth (then SKA-2 is descoped), or **(b)** integrate a telehealth/Rx partner and rewrite age gate + checkout. I cannot pick this. **Will escalate as a CEO decision** — see §7.

### Phase 1 — Brand-system swap (storefront-only, mechanical)

Goal: ship a CaliLean-branded storefront on the existing infra without changing data or commerce flows.

1. **Tokens & theme rename** — copy `bluum-tokens.css` → `calilean-tokens.css`, rename all `--bluum-*` to `--calilean-*`, update `globals.css` import. Update `tailwind.config.js` `colors.bluum` → `colors.calilean`. Decide CaliLean palette with CMO before committing the new hex values; structurally the rename is a single search-and-replace.
2. **Font decision** — keep Switzer or swap. If keeping, move the `@import` out of CSS and into `next/font` via `app/layout.tsx` for self-hosting and FCP wins. If swapping, do this in the same PR.
3. **Component module rename** — `storefront/src/modules/bluum/` → `storefront/src/modules/calilean/`. Update the 8 import sites (listed in §3.1). Rename `BluumLogo` → `CaliLeanLogo` and replace SVG path data with the new mark.
4. **Class-name rename** — repo-wide replace `bluum-` → `calilean-` in JSX `className` strings. ~100+ occurrences; mechanical. Verify by `pnpm build` (Tailwind JIT will fail on missing classes).
5. **Copy pass** — owned by CMO. Inventory in §3.4 is the working surface. Replace: `layout.tsx` title, homepage meta, hero, lab-banner, announcement-bar, value-props (with CaliLean's tester names), age-gate, FAQ (incl. support email), trust badges, footer disclaimer + copyright, store template subhead, research-disclaimer.
6. **Domain & email** — pick CaliLean support email (e.g. `hello@calilean.bio`); update footer + FAQ. Update README, CONTRIBUTING, CLAUDE.md.

Phase 1 is shippable in 2-3 engineer-days assuming palette + copy are pre-decided.

### Phase 2 — Catalog & assets migration

7. **Image hosting migration** — host product images on MinIO (already configured) or a new CDN. Rewrite `products-seed.json` URLs from `bluumpeptides.com/cdn/...` → new host. Strip the two embedded report links inside descriptions. Update `next.config.js:35` `remotePatterns` to allow the new host **and remove** `bluumpeptides.com`.
8. **Seed script** — rename SKU prefix `BLUUM-` → `CALILEAN-` in `scripts/seed-products.py:180`; update header comment + print line. Re-seed a clean dev database to validate.
9. **SEO scaffolding** — add `app/sitemap.ts`, `app/robots.ts`, `app/manifest.ts`, an OG image route, and JSON-LD on PDPs. New work, not a rename.

### Phase 3 — Loose ends

10. Inventory check in cart (SKA-6). Independent of brand.
11. Password-update flow.
12. Email template branding (CaliLean palette + footer).
13. Production CORS origins added to env templates / Railway.
14. `.gitignore` expansion; LICENSE file; CONTRIBUTING update.
